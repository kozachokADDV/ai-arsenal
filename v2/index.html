💰 Финансовый конвейер

Скрин квитанции в рабочей группе → Vision/OCR извлекает 7 полей (сумма / отправитель / получатель / № счёта / банк / назначение / дата) → детерминированная проверка маршрута лево/право → подтверждение каждой записи → Google Sheets. Голос работает через тот же конвейер (STT).

👤 Два канала клиентов

Личные клиенты — userbot отвечает «как Виталий» (voice-style + знания Рекламастер). Кто пишет боту — бренд-бот @Reclamaster_nr_bot. Оба канала ведут к одному мозгу Claude Code. Брифинг новых — LLM-интервью, а не форма.

🛡️ Анти-тихие-ошибки

Главный принцип: деньги не записываются с тихими ошибками. Подтверждение каждой записи + детерминированная страховка маршрута + confidence-gate. Всё «наружу» (email, ответы клиентам) — только после подтверждения.

// FOUNDATION · STRUCTURE

Фундамент: структура и память

В мае структура реорганизована под портфель бизнесов, а память агентов переведена на единую конвенцию «университета» — claims / verdicts / site-profiles, которые читаются на старте каждой сессии.

📁 Структура портфеля

Единая база Claud/: businesses/{reclamaster, m-stores, consulting}/, shared/agents/, tools/, _system/, archive/. Старые папки переехали в архив, ничего не теряется.

🎓 Университет памяти

Model C+++: глобальный _system/university/INDEX.md (claims · verdicts · inbox) + локальная память каждого бизнеса и агента (research · audits · learnings · site-profiles).

🔁 Цикл обучения

В конце сессии — апдейт памяти: ошибки которых избегать, паттерны что сработали, новые правила. Append-only, старые уроки не удаляются.

🧭 Глобальные правила

Единый ~/.claude/CLAUDE.md: язык, краткость, безопасность (GUI quit saving no, не /tmp), финансовая дисциплина (SSOT, порог ±300 ₴), «заточенный топор».
